# Topwar의 데미지 계산 공식

공식적으로 Topwar에서는 데미지 계산 공식을 공개한 적이 없습니다.  
일반적인 게임의 계산 방식과 과거 가장 게임사의 정보를 많이 소개하던 유튜버 `T-me`와 `Nass`의 정보를 토대로 추리하여 작성하는 글임을 미리 밝힙니다.

탑워의 데미지 계산 방식에 대한 추측 공식은 다음과 같습니다.

```
최종 피해량
=
기초 공격력
× [1
   + 2 × (병종 특화 공격력% + 장비 공격력% + 부품 공격력%)
   + 영웅 WAR 공격력% × 1.3
   + 기타 공격력%
  ]
× [1
   + 일반 데미지 증가%
   + 영웅 데미지 증가%
   + 상성 데미지 증가%
  ]
× (1 + 영웅 공격력%)
× 타격 배율
× (1 + 취약%)
× 스택 수
÷ [1
   + 상대 일반 방어력 증가%
   + 상대 상성 방어력 증가%
  ]
× (1 - 파괴불가 데미지 감소%)
```

간단하게 살펴보면
- 공격자 : 유닛의 기본 공격 데미지에 공격자의 공격력(atk), 데미지증가(dmg+)를 더하거나 곱한다.
- 방어자 : 공격자의 데미지를 방어자의 데미지감면(dmg-)과 방어도(def)로 감소시킨 뒤 생명력에서 차감한다.

더 간단하게 살펴보면 다음과 같습니다.

```
최종데미지 = (공격 데미지 합계) - (방어로 감소된 데미지)
```

하지만 간단하게 살펴보면 어느것이 어떤 효율을 가지고 있는지 알 수 없기 때문에 각각의 항목들에 대해서 알아보겠습니다.

## 기초 공격력 배율을 알아낸 과정

위 표에서 1~80레벨 구간은 확인 가능한 `Base ATK` 표를 기준으로 계산했습니다.  
반면 81레벨 이후에는 Base ATK 값이 직접 정리되어 있지 않아, 실제 전투 피해량을 이용해 역산했습니다.

### 1. 실험 조건 고정

레벨별 피해량을 비교할 때는 유닛 레벨을 제외한 모든 조건을 동일하게 유지했습니다.

- 같은 공격자
- 같은 영웅 편성
- 같은 장비와 파츠
- 같은 장식
- 같은 스택 수
- 같은 상대
- 같은 버프·디버프
- 같은 첫 타격
- 유닛 레벨만 변경

이 조건에서는 공격력, 데미지 증가, 방어력, 취약, 영웅 스킬 등의 계수가 모두 고정됩니다.

따라서 최종 피해량은 기초 공격력에 비례한다고 볼 수 있습니다.

\[
Damage_L = BaseATK_L \times K
\]

여기서 \(K\)는 이번 실험에서 고정된 나머지 전투 계수의 곱입니다.

---

### 2. 처음에는 레벨 성장률을 추정했다

초기에는 90레벨 이후 피해량이 레벨마다 약 5%씩 증가하는 것처럼 보였습니다.

예를 들면 다음과 같습니다.

| 레벨 | 측정 피해량 |
|---:|---:|
| 90 | 91B |
| 91 | 95.5B |
| 92 | 100B |
| 93 | 107B |
| 94 | 113B |
| 95 | 118B |
| 96 | 125B |
| 97 | 131B |
| 98 | 140B |
| 99 | 147B |
| 100 | 156B |
| 101 | 164B |

이 값만 보면 레벨당 약 5% 전후로 증가하는 지수 성장처럼 보입니다.

하지만 73레벨 피해량이 `15B`라는 점을 함께 비교하자 단순한 고정 성장률로는 전체 구간을 설명할 수 없었습니다.

따라서 다음 가설은 기각했습니다.

```text
최종 피해량 = 이전 레벨 피해량 × 일정 성장률
```

레벨이 직접 피해 보정으로 들어가는 것이 아니라, 레벨마다 별도의 `Base ATK` 값이 존재한다고 보는 편이 더 자연스러웠습니다.

---

### 3. 알려진 Base ATK와 실측 피해량을 비교했다

확인 가능한 표에서 다음 값을 기준점으로 사용했습니다.

| 레벨 | Base ATK | 실제 피해량 |
|---:|---:|---:|
| 73 | 225,000,000 | 약 15B |
| 79 | 550,000,000 | 36.5B |
| 80 | 650,000,000 | 43.2B |

각 레벨의 `피해량 ÷ Base ATK`를 계산하면 다음과 같습니다.

#### 79레벨

\[
\frac{36.5B}{550M}
\approx66.36
\]

#### 80레벨

\[
\frac{43.2B}{650M}
\approx66.46
\]

#### 73레벨

\[
\frac{15B}{225M}
\approx66.67
\]

73레벨은 피해량이 소수점 없이 `15B`로 표시되어 오차가 더 크지만, 세 값 모두 약 `66.4배` 부근에 모입니다.

따라서 동일한 전투 조건에서는 다음 관계가 성립한다고 판단했습니다.

\[
Damage \approx BaseATK \times 66.4
\]

여기서 `66.4`는 게임 전체의 고정 상수가 아니라, 해당 실험에서 사용한 공격자와 방어자의 모든 전투 계수를 합친 값입니다.

---

### 4. 79레벨 피해량을 사전에 예측했다

80레벨 피해량과 표의 Base ATK를 기준으로 79레벨 피해량을 예측했습니다.

\[
43.2B
\times
\frac{550M}{650M}
\approx36.55B
\]

예상값:

```text
약 36.5B
```

실제 측정값:

```text
36.5B
```

예상값과 실제값이 일치했습니다.

이 결과를 통해 다음 내용을 확인했습니다.

```text
동일 조건에서 최종 피해량은 Base ATK에 정비례한다.
```

즉 별도의 레벨 보정 계수를 추가하지 않아도 됩니다.

### 5. 81레벨 이후 배율을 역산했다

80레벨을 기준점으로 사용했습니다.

- 80레벨 Base ATK: `650,000,000`
- 1레벨 Base ATK: `1.25`
- 80레벨의 1레벨 기준 배율:

\[
\frac{650,000,000}{1.25}
=
520,000,000
\]

동일한 전투 조건에서는 피해량 비율과 Base ATK 비율이 같으므로, 81레벨 이후의 레벨 1 기준 배율은 다음 식으로 계산했습니다.

\[
LevelRatio_L
=
\frac{Damage_L}{Damage_{80}}
\times
LevelRatio_{80}
\]

실제 값으로 쓰면 다음과 같습니다.

\[
\boxed{
LevelRatio_L
=
\frac{Damage_L}{43.2B}
\times
520,000,000
}
\]

### 8. 최종 결론

이번 실험을 통해 다음과 같이 정리할 수 있습니다.

```text
유닛 레벨
→ 최종 피해량에 별도 계수로 직접 적용되는 것이 아님

유닛 레벨
→ 해당 레벨의 Base ATK를 결정

동일한 전투 조건
→ 최종 피해량은 Base ATK에 정비례
```

따라서 기초 공격력 항목은 다음처럼 처리하는 것이 가장 자연스럽습니다.

\[
\boxed{
BaseATK = BaseATKTable[UnitLevel]
}
\]

그리고 레벨별 피해량 관계는 다음과 같이 표현할 수 있습니다.

\[
\boxed{
\frac{Damage_A}{Damage_B}
=
\frac{BaseATK_A}{BaseATK_B}
}
\]

### 9. 주의사항

81레벨 이후의 배율은 공식적으로 공개된 Base ATK 값을 직접 사용한 것이 아니라, 실제 전투 피해량으로 역산한 추정값입니다.

또한 게임 화면의 피해량은 `B` 단위로 축약 표시되므로 다음 오차가 포함될 수 있습니다.

- 소수점 반올림
- 소수점 버림
- 내부 정수 피해량과 표시 피해량의 차이

따라서 81레벨 이후 수치는 정확한 내부 테이블 값이라기보다, 현재 실험에서 얻은 가장 근접한 추정치로 보는 것이 안전합니다.
