# Top War 전투 공식 실측 분석 총정리

> 이 문서는 게임사가 공개한 공식이 아니라, 동일 조건에서 능력치 하나만 바꾸는 방식으로 수집한 실측 자료를 기반으로 한 역산 결과입니다.  
> 화면의 `B`, `T` 축약값보다 리플레이의 원본 정수 피해량을 우선하며, 서로 충돌하는 과거 결과가 있을 때는 최신 원본 정수 실험을 우선합니다.

## 2. 실험 방법과 원본 피해 추출

화면 피해는 `19.3B`, `340B`처럼 축약되므로 정밀 역산에 부적합합니다.  
리플레이의 `onFMessage_NBattle_UnitAction` 이벤트에서 다음 조건을 사용했습니다.

```js
data.action === 3
data.data.lifeBarAnimIsDamage === true
```

핵심 필드:

```js
const damage = Math.abs(data.args[1]);
const hitType = data.args[2];
const referenceValue = data.args[11];
const skillId = data.skillConfig?.id;
```

현재 관측:

| 필드 | 의미 |
|---|---|
| `args[1]` | 최종 피해량의 음수 표현 |
| `args[2] = 1` | 일반타 |
| `args[2] = 4` | 치명타 |
| `args[11]` | 공격력·DMG+ 변경에는 고정된 참조값. 치명타에서는 1.92배가 되었으나 정확한 내부 명칭은 미확정 |
| `skillId = 1011022` | 이번 정밀 비교에 사용한 동일 공격 스킬 |
| `fromPos = 0`, `toPos = 0` | 동일 슬롯 간 비교 |

정밀 실험에서는 다음을 고정했습니다.

- 같은 공격자와 방어자
- 같은 유닛 레벨과 출전 수
- 같은 영웅 및 스킬
- 같은 공격 슬롯과 피격 슬롯
- 같은 `skillId`
- 같은 일반타 또는 같은 치명타끼리 비교
- 새 스펙 적용 후 새 전투의 리플레이 사용

## 3. 현재까지의 통합 공식

현재까지 검증된 구조를 합치면 다음과 같습니다.

```text
최종 피해
=
BaseATK[유닛 레벨]
× 출전 유닛 수
× 일반 공격력 블록
× 영웅 전투 스킬 공격력 블록
× 데미지 증가 블록
× 타격·상성·취약·스킬 계수
÷ DEF 블록
÷ DMG- 블록
× 치명타 계수
```

수식으로 표현하면:

\[
D =
B_L
\times N
\times A
\times H_A
\times G
\times K
\div F_{DEF}
\div F_{DMG-}
\times C
\]

- \(B_L\): 레벨별 Base ATK
- \(N\): 출전 유닛 수
- \(A\): 일반 공격력 블록
- \(H_A\): 영웅 전투 스킬 공격력 블록
- \(G\): 데미지 증가 블록
- \(K\): 이번 문서에서 생략한 타격·상성·취약·특수 스킬 계수
- \(F_{DEF}\): DEF 방어 블록
- \(F_{DMG-}\): 데미지 감면 블록
- \(C\): 치명타 계수

### 현재 정밀 실험 기준의 변화식

현재 기준 피해를 \(D_0=340,994,470,531\)로 두면:

\[
D_{new}
=
D_0
\times
\frac{B_{L,new}}{B_{L,0}}
\times
\frac{N_{new}}{N_0}
\times
\frac{A_{new}}{66.64791866}
\times
\frac{H_{A,new}}{H_{A,0}}
\times
\frac{G_{new}}{12.40225393}
\times
\frac{F_{DEF,0}}{F_{DEF,new}}
\times
\frac{F_{DMG-,0}}{F_{DMG-,new}}
\times
\frac{C_{new}}{C_0}
\]

이 비율식은 서로 다른 전투의 절대 상수를 억지로 추측하지 않고, 한 항목만 바꿨을 때의 피해량을 계산하는 데 가장 안전합니다.

# 4. 레벨별 Base ATK 실험

## 4.1 실험 조건

유닛 레벨만 변경하고 공격자, 방어자, 영웅, 장비, 부품, 스킬, 출전 수와 버프를 고정했습니다.

결과:

\[
\boxed{Damage \propto BaseATK[Level]}
\]

즉 레벨이 최종 피해에 별도 보정 계수로 곱해지는 것이 아니라, 레벨이 `BaseATK` 테이블 값을 결정합니다.

현재처럼 리플레이 원본 정수 피해량을 확보하면 이 표도 더 정밀하게 갱신할 수 있습니다.

# 5. 출전 유닛 수

동일 조건에서 출전 유닛 수는 피해량과 총 생명력에 정확히 선형 적용됩니다.

\[
\boxed{Damage \propto N}
\]

\[
\boxed{TotalHP \propto N}
\]

따라서 현재 출전 수가 \(N\)일 때 출정 `+1`의 공격 및 총 생명력 증가율은 각각:

\[
\boxed{\frac{1}{N}\times100\%}
\]

입니다.

예를 들어 출전이 400이라면:

```text
출정 +1
→ 피해 +0.25%
→ 총 생명력 +0.25%
```

공격과 총 생명력이 동시에 증가한다는 점이 중요합니다.

## 6.2 장식 전체 공격력

장식 전체 공격력 60%p 제거:

```text
340,994,470,531
→ 337,924,657,003
감소량: 3,069,813,528
```

장식 전체 공격력 102%p 제거:

```text
340,994,470,531
→ 335,775,787,534
감소량: 5,218,682,997
```

1%p당 감소량:

```text
60% 실험: 3,069,813,528 ÷ 60 = 51,163,558.800
102% 실험: 5,218,682,997 ÷ 102 = 51,163,558.794
```

두 값이 정수 오차 수준에서 일치합니다.

\[
\boxed{장식\ 전체\ 공격력\ 1\%p \Rightarrow 피해\ 약\ 51,163,558.8\ 증가}
\]

현재 공격력 블록은:

\[
A_0
=
\frac{D_0\times0.01}{장식ATK\ 1\%p당\ 피해}
=
\boxed{66.64791866}
\]

즉 현재 전투의 유효 일반 공격력 합계는 블록 표현으로 약:

\[
(A_0-1)\times100
=
\boxed{6564.791866\%}
\]

입니다.

이 수치는 화면의 `5143.7%`를 그대로 복사한 값이 아닙니다.  
파츠처럼 내부 가중치가 2배인 항목을 포함해 피해 변화로 역산한 **유효 공격력 합계**입니다.

## 6.3 파츠 공격력

파츠 공격력 100%p 제거:

```text
340,994,470,531
→ 330,761,758,771
감소량: 10,232,711,760
```

파츠 공격력 180%p 제거:

```text
340,994,470,531
→ 322,575,589,364
감소량: 18,418,881,167
```

1%p당 감소량:

```text
100% 실험: 102,327,117.600
180% 실험: 102,327,117.594
```

장식 1%p당 피해의 정확히 2배입니다.

\[
\boxed{파츠\ 공격력\ 1\%p = 장식\ 전체\ 공격력\ 2\%p}
\]

여기서 2배는 최종 피해가 두 배가 된다는 뜻이 아닙니다.  
일반 공격력 블록에 들어갈 때 파츠 수치가 2배 가중된다는 뜻입니다.

현재 확인된 부분만 적으면:

\[
A
=
1+
\frac{ATK_{일반}+2ATK_{파츠}+\cdots}{100}
\]

## 6.4 장식 복구 검증

장식 제거 후 다시 착용했을 때:

```text
damage: 340,994,470,531
args[11]: 134,144,010
args[2]: 1
```

기준값과 정수 단위까지 완전히 일치했습니다.  
따라서 장식 60%와 102% 제거 결과는 재현 가능한 실험입니다.

## 6.5 제외한 300% 실험

파츠 공격력 300% 제거라고 기록된 초기 실험에서는 피해가 `274,767,418,867`로 나왔습니다.

그러나 이후 정밀 실험으로 확인한 파츠 계수에 따르면 예상값은 약:

```text
310,296,335,251
```

이어야 합니다.

따라서 해당 초기 전투는 파츠 공격력 이외의 조건이 함께 바뀐 오염된 데이터로 판단하며, 정밀 공식의 근거에서는 제외합니다.

# 7. 영웅 전투 스킬 공격력

## 7.1 엘라 스킬 실험

테스트한 준비 스킬:

```text
전투 시작 시 육군 공격력 115.3% 증가
```

전속 스킬 효과 향상 상태:

| 스킬 효과 향상 | 첫 타격 피해 |
|---:|---:|
| 0% | 452B |
| 3% | 459B |
| 6% | 466B |

관측 배율:

```text
3% 향상: 459 / 452 ≈ 1.01549
6% 향상: 466 / 452 ≈ 1.03097
```

스킬 효과 향상 3%가 최종 피해를 그대로 3% 올리는 것은 아닙니다.

기본 스킬 공격력 증가율을 \(S\), 스킬 효과 향상을 \(E\)라고 하면 실측에 잘 맞는 구조는:

\[
\boxed{H_A=1+S(1+E)}
\]

엘라의 경우:

\[
S=1.153
\]

3% 향상 예상:

\[
\frac{1+1.153\times1.03}{1+1.153}
\approx1.01607
\]

```text
452B × 1.01607 ≈ 459.26B
실측: 459B
```

6% 향상 예상:

\[
\frac{1+1.153\times1.06}{1+1.153}
\approx1.03213
\]

```text
452B × 1.03213 ≈ 466.52B
실측: 466B
```

## 7.2 일반 공격력과 별도 블록 검증

일반 공격력을 변경하면서 스킬 효과 0%, 3%, 6%를 교차 테스트했습니다.

| 일반 공격력 상태 | 0% | 3% | 6% |
|---|---:|---:|---:|
| 기준 | 452B | 459B | 466B |
| 전체 공격력 4% 제거 | 451B | 458B | 465B |
| 전체 공격력 10% 제거 | 450B | 457B | 464B |
| 과거 300% 제거 기록 | 353B | 359B | 365B |

일반 공격력이 달라져도 스킬 효과 향상의 상대 배율은 거의 유지됐습니다.

따라서:

\[
\boxed{Damage\propto GeneralATKBlock\times HeroSkillATKBlock}
\]

영웅 전투 스킬 공격력은 화면 일반 공격력 합계에 더해지는 항목이 아니라, 별도 곱연산 블록으로 보는 것이 실측에 맞습니다.

> 이 스킬식은 엘라의 해당 준비 스킬에서 검증한 식입니다. 다른 영웅의 특수한 스킬이 모두 동일하다고 단정하지는 않습니다.

# 8. 데미지 증가

## 8.1 초기 화면 축약값 실험

다른 조건을 모두 고정하고 일반 데미지 증가만 바꿨습니다.

| DMG+ | 측정 피해 |
|---:|---:|
| 773.8% | 1.56T |
| 793.8% | 1.59T |
| 843.8% | 1.68T |

각 피해를 `1 + DMG+/100`으로 나누면 약 `0.178T` 부근으로 모였습니다.

따라서 초기 실험에서도:

\[
\boxed{Damage\propto1+\frac{DMG+}{100}}
\]

구조를 확인했습니다.

## 8.2 리플레이 정수 정밀 실험

DMG+ 50%p 제거 전후:

```text

50%p 제거: 327,247,192,316
```

비율:

\[

=
\frac{327247192316}{340994470531}
=
0.9596847474
\]

\[
\frac{G-0.5}{G}=r
\]

따라서:

\[
\boxed{G_0=12.40225393}
\]

유효 DMG+ 합계:

\[

=

보다 약 `118.425%p` 높습니다.

DMG+를 제거해도 `args[11]`은 `134,144,010`으로 고정됐으므로, 이 참조값에는 일반 DMG+가 포함되지 않았습니다.

# 11. 생명력과 영웅 스킬 생명력

## 11.1 일반 생명력 블록

영웅을 제외한 상태에서:

```text
전체 생명력 6278%
→ 전체 생명력 300%p 제거
→ 5978%
```

일반 생명력 블록은:

\[
\boxed{GeneralHPBlock=1+\frac{전체생명력}{100}}
\]

따라서 제거 전후 비율은:

\[
\frac{1+59.78}{1+62.78}
=
\frac{60.78}{63.78}
\approx0.952963
\]

입니다.

## 11.2 영웅 스킬 생명력은 별도 블록

제트 스킬 비교에서 기록된 두 대응 상태:

| 상태 | 화면 전체 생명력 | HP 기록 |
|---|---:|---:|
| 스킬 제외 | 6188% | 3.24T / 3.36T |
| 스킬 적용 | 6198% | 3.69T / 3.83T |

실제 HP 배율:

```text
3.69 / 3.24 ≈ 1.1389
3.83 / 3.36 ≈ 1.1399
```

화면 전체 생명력 10%p 차이만으로 예상되는 배율은:

\[
\frac{62.98}{62.88}
\approx1.00159
\]

에 불과하므로 실제 약 1.139배를 설명하지 못합니다.

따라서:

\[
\boxed{TotalHP
=
BaseHP
\times N
\times GeneralHPBlock
\times HeroSkillHPBlock}
\]

일반 생명력과 영웅 전투 스킬 생명력은 별도 곱연산 블록입니다.

영웅 스킬 기본 생명력 증가율을 \(S_H\), 스킬 효과 향상을 \(E\)라고 하면 당시 실측과 맞춘 구조는:

\[
\boxed{HeroSkillHPBlock=1+S_H(1+E)}
\]

입니다.

## 11.3 스킬 생명력 강화 화면 기록

생명력 스킬 강화 전후로 기록한 축약 HP 값:

| 강화 전 | 강화 후 | 증가율 |
|---:|---:|---:|
| 4.34T | 4.58T | 약 5.53% |
| 4.48T | 4.72T | 약 5.36% |
| 6.71T | 7.10T | 약 5.81% |

축약 표시 오차가 있지만 모두 비슷한 범위로 증가했습니다.  
이 값들은 원본 정수 HP가 아니라 화면 축약값이므로, 공격력 정밀 실험만큼의 소수점 정확도를 주장하지 않습니다.

과거 분석 중 제기됐던 `숨은 생명력 23~24%` 추정은 근거가 부족하여 폐기했습니다.  
현재 유지하는 결론은 **일반 생명력과 스킬 생명력이 별도 곱연산 블록**이라는 점입니다.

# 12. 질문 1: DMG+ 1%는 공격력 몇 %와 같은가?

\[
G=1+\frac{유효DMG+}{100}

\[
\frac{1}{A}\%

파츠 공격력 1%p의 최종 피해 증가율:

\[
\frac{2}{A}\%

DMG+ 1%p의 최종 피해 증가율:

\[
\frac{1}{G}\%
\]

따라서:

\[

=
장식ATK\ \frac{A}{G}\%p}

\[
\boxed{DMG+\ 1\%p
=
파츠ATK\ \frac{A}{2G}\%p}
\]

\[

\]

\[
\boxed{DMG+\ 1\%p
\approx 장식ATK\ 5.374\%p}
\]

\[
\boxed{DMG+\ 1\%p
\approx 파츠ATK\ 2.687\%p}
\]

# 14. 질문 3: DEF 장식 100개와 DMG- 장식 75개는 합리적인가?

```text
DEF 1%p ≈ DMG- 1.5509%p
```

=
\boxed{116.3개}
\]

입니다.

실제 DEF 장식 비용은 100개이므로 현재 구간에서는:

# 15. 질문 4: 출정 +1은 어느 정도의 스펙 가치인가?

현재 출전 수를 \(N\)이라고 하면:

\[
출정+1의 피해 증가율
=
출정+1의 총HP 증가율
=
\frac{100}{N}\%
\]

출정 +1과 같은 공격 옵션:

\[
\boxed{DMG+\ 환산=\frac{100G}{N}\%p}
\]

\[
\boxed{장식ATK\ 환산=\frac{100A}{N}\%p}
\]

\[
\boxed{파츠ATK\ 환산=\frac{50A}{N}\%p}
\]

출정 +1과 같은 방어 옵션:

\[
\boxed{DEF\ 환산=\frac{100F_D}{N}\%p}
\]

\[
\boxed{DMG-\ 환산=\frac{100F_R}{N}\%p}
\]

현재 공격·방어 블록을 기준으로 한 예시:

| 현재 출정 N | 출정 +1 공격/HP 증가 | DMG+ 환산 | 장식 ATK 환산 | 파츠 ATK 환산 | DEF 환산 | DMG- 환산 |
|---:|---:|---:|---:|---:|---:|---:|
| 350 | 0.2857% | 3.544%p | 19.042%p | 9.521%p | 0.501%p | 0.777%p |
| 370 | 0.2703% | 3.352%p | 18.013%p | 9.006%p | 0.474%p | 0.735%p |
| 383 | 0.2611% | 3.238%p | 17.402%p | 8.701%p | 0.458%p | 0.710%p |
| 400 | 0.2500% | 3.101%p | 16.662%p | 8.331%p | 0.438%p | 0.680%p |
| 450 | 0.2222% | 2.756%p | 14.811%p | 7.405%p | 0.390%p | 0.604%p |


예를 들어 출전 383일 때:

```text
출정 +1
→ 피해 +0.2611%
→ 총 생명력 +0.2611%
→ DMG+ 약 3.238%p와 같은 공격 증가
→ 장식 전체 공격력 약 17.402%p와 같은 공격 증가
→ 파츠 공격력 약 8.701%p와 같은 공격 증가
→ DEF 약 0.458%p와 같은 내구 증가
→ DMG- 약 0.710%p와 같은 내구 증가
```

중요한 점은 출정 +1이 위 옵션 중 하나만 제공하는 것이 아니라:

```text
공격력 증가 효과
+
총 생명력 증가 효과
```

를 동시에 제공한다는 것입니다.

피해와 총 HP의 단순 곱을 전투력 지표로 사용한다면:

\[
\left(\frac{N+1}{N}\right)^2-1
\]

만큼 증가하지만, 실제 전투 승률은 선공, 스킬 발동, 오버킬, 병력 손실 순서 등에 영향을 받으므로 이 곱을 공식 전투력으로 단정해서는 안 됩니다.

# 16. 현재 실전 계산기용 식

# 17. 데이터 품질과 확정 수준

## 높은 확정도

- 출전 수는 피해와 총 HP에 선형 적용
- 피해는 레벨별 Base ATK에 정비례
- 장식 전체 공격력은 일반 공격력 블록에 1배 가중
- 파츠 공격력은 같은 블록에 2배 가중
- DMG+는 `1 + 합계/100` 블록
- DEF와 DMG-는 서로 다른 곱연산 방어 블록
- 일반 공격력과 영웅 전투 스킬 공격력은 별도 곱연산
- 일반 생명력과 영웅 전투 스킬 생명력은 별도 곱연산
- 현재 치명타 계수는 1.92

## 조건부 확정

- DEF와 DMG-의 시작 상수 `1.1517`, `1.5729`는 현재 방어자 기준 유효값
- 유효 일반 공격력 합계 `6564.79%`와 유효 DMG+ 합계 `1140.23%`는 현재 공격자·상대·첫 타격 기준
- 엘라와 제트에서 확인한 영웅 스킬 증폭식이 모든 영웅에게 동일한지는 별도 확인 필요
- 81레벨 이상 Base ATK 역산값은 화면 축약 피해를 사용해 근삿값

## 공식에서 의도적으로 제외한 항목

# 18. 최종 요약

```text
[공격]

Base ATK는 레벨 테이블 값이며 피해에 정비례
출전 수는 피해에 정확히 선형 적용

장식 전체 공격력 1%p
→ 일반 공격력 블록 +0.01

파츠 공격력 1%p
→ 일반 공격력 블록 +0.02
→ 장식 공격력의 정확히 2배 가치

DMG+ 1%p
→ DMG 블록 +0.01

현재 구간:
DMG+ 1%p
≈ 장식 ATK 5.374%p
≈ 파츠 ATK 2.687%p
```

```text
[방어]

DEF와 DMG-는 별도 곱연산 블록

현재 DEF 60.2%, DMG- 114.7%:
DEF 1%p
≈ DMG- 1.551%p

장식 비용:
DEF 100개 vs DMG- 75개
→ 현재 구간에서는 DEF 장식이 약 16.3% 더 높은 재료 효율
```

```text
[출정]

출정 +1의 피해 증가율 = 100/N %
출정 +1의 총 HP 증가율 = 100/N %

출정 383 예시:
피해 +0.2611%
총 HP +0.2611%
DMG+ 약 3.238%p 상당의 공격 효과
DEF 약 0.458%p 상당의 방어 효과
두 효과를 동시에 제공
```
